Reassured App API Docs
Notifications API

This API is used in the form of a get request

Required for all endpoints:
    email - the reassured email address you use to sign into the app
	password - the password that you use to sign into the app
	
Notification types:
	everyone:
	    Sends the notification to all devices that are registered
	team:
	    Sends the notification to all members of the specified team
	single:
		Sends the notification to a single user
		
Additional fields:
	everyone:
	    notify:
			travel - Sends an alert to all users. Text is "$user_name, check your route before you travel. There are reported incidents today"
			calendar:
			    timeframe:
					today - Sends an alert to all users. Text is "$user_name, there is an event in the calendar for today"
					future - Sends an alert to all users. Text is "$user_name, there are new events in the calendar"
			social - Sends an alert to all users. Text is "$user_name, there are new MyReassured posts!"
	team:
		notify:
			late - sends an alert to everyone in a specific team_id. Text is "$affected_user is running late today."
				team_id - the team_id of the affected user.
				user_id - the user id of the affected user.
			meeting - sends an alert to everyone in a specific team_id. Text is "$team_manager has called a team meeting."
				team_id - the team_id of the affected user.
			lockout - sends an alert to everyone in a specific team_id. Text is "$affected_user is locked out, please let them in."
				team_id - the team_id of the affected user.
				user_id - the user id of the affected user.
	single:
	    notify:
			meeting - Sends an alert to that user. "$user_name, you have a meeting scheduled today at $time"
				user_id - the user_id
				time - unix timestamp of the meeting

	Example 1: sends a travel alert to all users
